Project:
Retrieval-Augmented Generation (RAG) using LangChain + Mistral AI + ChromaDB

-------------------------------------------
1. What is RAG?
-------------------------------------------

RAG (Retrieval-Augmented Generation) is a technique that allows an LLM to answer questions using your own documents instead of relying only on its training data.

Flow:

User Question
      ↓
Embedding Model
      ↓
Vector Database (ChromaDB)
      ↓
Relevant Chunks Retrieved
      ↓
LLM (Mistral)
      ↓
Final Answer

----------------

No module named torch

API Key Error

→ Check .env file

Empty Retrieval

→ Documents not embedded
→ Wrong collection
→ Wrong persist directory

20. Best Practices
-------------------------------------------

✔ Use RecursiveCharacterTextSplitter

✔ Chunk Size = 800–1200

✔ Chunk Overlap = 150–250

✔ Store ChromaDB persistently

✔ Cache embeddings

✔ Use .env for API keys

✔ Keep prompt simple

✔ Return source documents

✔ Log errors

✔ Validate uploaded files
